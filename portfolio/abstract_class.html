Abstract class:-

If a class implementation is not completed then we declear that class as abstract and also the object creation is not allowed.We can also say it is a concept of partially implementation of class.

Important Facts:-

 If a class contains at least one abstract method then that class must be abstract.
 We also declear a class as abstract that does not contains any abstract method.
 Abstract class also contains zero number of abstract method.
 Child class is responsible for implementation of abstract method.
 The implementation of all abstract method is in the child class.

eg:-

abstract class Test{
public abstract void m1();
public abstract void m2();
}
class Subtest extends Test{
public void m1(){
System.out.println("abstract method m1");
}
public boid m2(){
System.out.println("abstract method m2");
}
}

eg:-

abstract class Vehical{
public abstract int noOfWheel();
}
class Bus extends Vehical{
public int noOfWheel(){
return 6;
}
}
class Auto extends Vehical{
public int noOfWheel(){
return 3;
}
}
class Test{
public static void main(String[] args){
Bus b=new Bus();
System.out.println(b.noOfWheel());
Auto a=new Auto();

}
