Class Modifiers:-

The class modifiers basically describe the behaviour of the class.

Case-I(public)

public class Test{
block of codes
}

 When the class modifier is public then we can user anywhere either within the class or outside the class.

Case-II(default)

class Test{
block of codes
}

 When the class modifier is deafult then that class is only used within the package.

Case-III(abstract)

abstract class Test{
block of codes
}

 When the class modifier is abstract then the object creation is not allowed.Abstract mean that the defination of the class is not clearly defined.

Case-IV(final)

final class Test{
block of codes
}

 When the class modifier is final then the child class creation is not allowed but when the class is not final then child class creation is allowed.

Important Facts:-

 Based on these modifiers we specify the rules to the JVM that means the characteristics of the class.

 In the higher class decleration only public,deafult,final are allowed, but when we use inner class then private, protected, & static are alowed including public, default, abstract & final.

Feasibility:-

public+abstract+default+final(For high level class)
public+abstract+default+final+private+protected+static(For inner class)

Member Modifiers:-

public,private and protected are the member modifiers.

public:-

package p1;
class Test{
public void m1(){
System.out.print("Hello java");
}
}

package p2;
import p1.A;
public class B{
public static void main(String[] ar){
A a=new A();
a.m1();
}
}

 When the method or variables is public then we can user anywhere either within the class or outside the class also within the package or outside the package.

private:-

class Test{
private void m1(){
System.out.println("private modifier");
}
}
class Test{
public static void main(String[] ar){
A a=new A();
a.m1();
}
}

compile time error occurs because m1() method is decleared as private. So, it only can be accessible with in the same class.

 When the class modifier is private then it is only accessiable in the same class or within the class.

protected:-

package p1;
class A{
protected void m1(){
System.out.println("protected class");
}
}

package p2;
import p1.A;
class B entends A{
public static void main(String[] ar){
B b=new B();
b.m1();
}
}

 When the variables and methods are decleared as protected then it is used within the same package anywhere but outside the package it is used in the child class and only called by child refrence.
